1. Conversation Overview
The discussion focuses on the architectural evolution of the SimpleLLM system, transitioning it from a basic "keyword matching" logic to a sophisticated "semantic interpretation" engine. The project is structured into three development phases:

Vector Embedding Integration (Completed),
Intent Mapping (Current Phase),
Contextual Layering (Pending).
The primary goal of the current phase is to decouple raw user input from specific game rules by introducing an "Intent" layer, allowing for greater flexibility in how the system interprets player actions.

2. Active Development
Development is currently centered on Stage 2: Intent Mapping. The focus is on refactoring StoryGameEngine.cs to implement an intermediate step between raw input processing and rule execution. Instead of mapping a normalized string directly to a rule based on a similarity score, the system will now identify a high-level "Intent" (e.g., "Interact", "Move") before evaluating specific action rules. This approach aims to improve the robustness of interaction handling when multiple phrasing variations result in the same intended game outcome.

3. Technical Stack
Language/Runtime: C# / .NET.
Inference Engine: ONNX (utilized for local embedding generation).
Mathematics/Algorithms: Cosine Similarity (via TensorPrimitives), SIMD-accelerated operations, and Vector Embeddings.
Key Concepts: Semantic Scoring (0.0–1.0), Similarity Thresholds (e.g., 0.58), and Vector Clustering for Intent mapping.
4. File Operations
SimpleLLM.Library/LocalVectorEngine.cs: Implements IEmbeddingService. It provides the backend logic for ONNX inference and calculates cosine similarity between user input and game triggers.

HandleInput(string playerInput): Processes user commands.
GetMatchScore(string normalizedInput, IReadOnlyList<string> verbs): Calculates the semantic similarity score for a given action.
